## 失败分诊 · USE-OS-01-001 · runner.os 返回值与文档声明的平台支持一致

**判定结果**: FAIL
**失败断言**:
- positive/run_logs: contains `"os=Linux"` — actual log contains `"os=linux"`（小写 l）

**根因初判**: 产品bug
**责任人**: 平台方

**证据**:

- **Job 日志全量**:
  ```
  === JOB: test runner.os value (status=COMPLETED) ===
  [2026/07/23 22:44:21.055 GMT+08:00] [INFO] Job(1529982528146771968_1529982528121606151) duration check: true
  No shell specified, using platform default: default-bash
  ::debug::Script file created: /home/slave1/runner/workers/0.0.4.4.version/_temp/d1f6d39c-422f-4369-9656-1e6baa3f91ed.sh
  ::debug::Executing: bash -e /home/slave1/runner/workers/0.0.4.4.version/_temp/d1f6d39c-422f-4369-9656-1e6baa3f91ed.sh
  os=linux
  ```

- **预期行为**（Phase 01 文本用例）:
  - 前置条件: Linux Runner 可用
  - 操作步骤: 在 workflow 中输出 ${{ runner.os }}
  - 预期结果: 返回 Linux；若实际仅支持 Linux，文档必须标注 Windows/macOS 待定

- **实际行为**:
  - `runner.os` 返回字符串 `linux`（全小写），非规格声明的 `Linux`（首字母大写）
  - **失败传导链**: 平台 → `runner.os` 返回 `linux`（lowercase） → 日志输出 `os=linux` → 测试断言 contains `"os=Linux"` 匹配失败

- **测试 YAML 与规格精确对照**:
  - **测试 YAML** 中 `test-os` job 的 `echo runner os` step:
    ```yaml
    steps:
      - name: echo runner os
        run: |
          echo "os=${{ runner.os }}"
    ```
  - **GitCode 规格** `syntax-reference/context.md` 第209-228行:
    ```markdown
    ### runner 上下文
    
    ```json
    {
      "os": "Linux",
      "arch": "X64",
      "name": "Atomgit Actions 2",
      "tool_cache": "/opt/hostedtoolcache",
      "temp": "/home/runner/work/_temp"
    }
    ```
    
    | 属性 | 说明 |
    |------|------|
    | `runner.os` | 操作系统：`Linux`, `Windows`, `macOS` |
    ```
  - **逐项映射**:
    - 测试 `${{ runner.os }}` → 规格声明返回 `Linux`（首字母大写）
    - 实际运行时返回 `linux`（全小写），与规格不一致
    - 差异：规格说 `Linux`/`Windows`/`macOS` (PascalCase)，平台返回全小写

- **环境前置条件验证**: Linux Runner (dedicate-hosted, x64, large)，workflow_dispatch 触发

**置信度**: 高（日志直接输出 `os=linux`；规格明确列出 `Linux`、`Windows`、`macOS`）

**影响**:
- **阻塞性**: 🟡 如果用户依赖 `${{ runner.os == 'Linux' }}` 做操作系统条件判断，条件恒为 false
- **静默性**: 🟡 runner.os 返回了值，但值与文档声明不一致
- **影响面**: 🟡 所有依赖 `runner.os` 做条件判断或矩阵选择的 workflow
- **综合**: platform 返回的 os 值大小写与规格不一致，破坏条件表达式
- **是否有规避手段**: 是（使用 `${{ runner.os == 'linux' }}` 小写比较；但用户只能通过试错发现）

**建议**:
- 平台方应统一 `runner.os` 返回值的大小写格式与规格一致（`Linux`、`Windows`、`macOS`）
- 或若全小写是有意设计，则需同步更新 `syntax-reference/context.md` 第222-223行的 runner.os 说明
- 同时建议补全文档：若实际仅支持 Linux，需附加"当前仅 Linux 可用，Windows/macOS 待定"标注
