## 失败分诊 · REL-K8S-01-045 · 自托管 K8s Runner 弹性伸缩——min=1/max=1 时并发 3 个 jobs 应排队执行

**判定结果**: FAIL
**失败断言**: 正向/pod_count expected=1 actual=N/A; 正向/max_concurrent_jobs expected=1 actual=N/A

**根因初判**: 环境/Harness（log 仅 1 行——runner 可能不可用或 job 被拒绝调度）
**责任人**: Phase 02

**证据**:

- **Job 日志全量**:
  ```
  === JOB: K8s runner scaling test (status=FAILED) ===
  [2026/07/23 22:30:25.434 GMT+08:00] [INFO] Job(1529979023189090304_1529979023155535879) duration check: true
  ```

- **预期行为**（Phase 01 文本用例）:
  - 前置条件: 仓库具备 K8s Runner 使用权限; K8s Runner 组配置 min=1 max=1
  - 操作步骤: 并发触发 3 个 jobs 到 K8s Runner 组
  - 预期结果: Runner Pod 数始终=1; 3 个 jobs 顺序执行; 不应创建 2 个及以上 Pod

- **实际行为**:
  - 仅 1 个 job instance 出现在日志中，且仅 2 行——`duration check: true` 之后无任何后续内容
  - Job status=FAILED，但没有任何 step 执行日志（甚至没有 `Using platform default: default-bash`）
  - 说明 job 在 runner 分配或启动阶段即失败——可能是 K8s runner 组不可用或 self-hosted runner 未就绪
  - **失败传导链**: runner 未分配/启动失败 → 无 step 执行 → job FAILED → 所有断言不满足

- **测试 YAML 与规格精确对照**:
  - **测试 YAML** 中 `test` 的 runs-on:
    ```yaml
    runs-on: [self-hosted, kubernetes, small]
    ```
  - **测试 YAML** 中 `test` step:
    ```yaml
    - name: sleep step
      run: |
        sleep 10
    ```
  - **GitCode 规格** `writing-pipelines/configure-jobs.md` 第 57-65 行:
    ```yaml
    runs-on:
      type: self-hosted
      group: my-runner-group
      labels:
        - linux
        - x64
        - gpu
    ```
  - **GitCode 规格** `runner-management/selecting-runner-labels.md` — 自托管 K8s Runner 标签
  - **逐项映射**: `runs-on: [self-hosted, kubernetes, small]` → 匹配 self-hosted 标签三段式格式; `sleep 10` → 简单测试步骤。测试 YAML 语法与规格一致。但环境条件（K8s Runner 组 min=1/max=1）未被验证——若 K8s runner 没有可用 Pod，job 无法被调度。

- **环境前置条件验证**: K8s runner 可能不可用——job 在日志首行 `duration check: true` 后即结束，无任何 step 执行记录

**置信度**: 中（runner 不可用可能性大，但日志信息极少——仅 2 行，无法定位确切原因）

**影响**:
- **阻塞性**: 🔴阻塞 — K8s runner 不可用导致用例完全无法执行
- **静默性**: 🟡中等 — 仅 `status=FAILED` 但无详细错误信息
- **影响面**: 🟡同 runner — 影响所有 K8s runner 用例
- **综合**: K8s self-hosted runner 可能未正确注册或 Pod 不可用
- **是否有规避手段**: 是（确认 K8s Runner 组 Pod 状态，手动注册 K8s runner 后再执行）

**建议**:
- 检查 K8s runner 注册状态和 Pod 数量
- 确认 test harness 在 Phase 02 中是否正确配置了 K8s runner 环境
- 如果 K8s runner 不可用，跳过此用例或标记为 ENV_NOT_READY
