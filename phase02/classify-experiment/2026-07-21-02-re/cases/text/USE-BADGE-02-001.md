用例 ID:   USE-BADGE-02-001
维度标签:   [usability]
维度:      易用性
优先级:    P2
溯源意图:  INTENT-USE-027
标题:      Workflow 状态徽标 badge SVG 的可用性、分支过滤与状态同步及时性

前置条件:
  - 目标仓库存在已配置的工作流
  - 该工作流已有至少一次运行记录

操作步骤:
  1. 获取该工作流的 badge URL，确认返回有效 SVG
  2. 检查 SVG 中的状态颜色/文本是否与最近运行结果一致
  3. 触发一次新的运行并使其失败（或等待自然状态变更）
  4. 在合理时间窗口内再次请求 badge URL，确认状态已刷新为失败
  5. 若平台支持分支过滤，测试带分支参数的 badge URL 行为

预期结果:
  - badge URL 返回有效 SVG 且状态与指定 workflow 最近运行一致
  - 运行结果变更后 badge 在合理时限内刷新（不超过 5-10 分钟量级）
  - 若支持分支过滤，参数格式应文档化；不支持则应给出明确说明

验证点:
  - [正向] badge URL 返回 SVG，图形/文本状态与最近运行结果一致
  - [负向] 运行结果变更后 badge 不应长期显示旧状态（如失败运行后仍显示绿色通过徽标）
  - [非功能] 是否支持按分支过滤；workflow_name 匹配规则（文件名 vs name 字段）的文档清晰度
  - [非功能] badge 嵌入文档的清晰度、分支过滤支持的「够用程度」（主观判据）
