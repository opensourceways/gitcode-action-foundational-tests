标题:      OBS 下载完整链路——基本下载/不存在对象/变量路径/权限/大文件/端到端联调

前置条件:
  - 仓库配置了 OBS 下载 workflow
  - OBS bucket 中存在已知测试对象
  - 已知网络偶发失败 (#15)、变量路径问题 (#16)

操作步骤:
  1. 配置 workflow 使用 `uses: official_obs_download`，指定 bucket+object key 下载到本地
  2. 验证下载文件内容与上传源一致
  3. 指定不存在的 object key，观察报错
  4. 使用 `${{ env.DOWNLOAD_DIR }}` 等变量指定下载路径
  5. 尝试下载到无写权限的路径，观察报错
  6. 下载大于 100MB 的文件，验证稳定性
  7. 先上传再下载，验证端到端内容一致性（与 `official_obs_upload` 联调）

预期结果:
  - 基本下载成功，文件内容正确
  - 下载不存在对象时报错清晰（非泛化 failure）
  - 变量路径正确解析
  - 无写权限时给出明确权限错误
  - 大文件下载不超时
  - 端到端上传→下载内容一致

验证点:
  - [正向] 下载文件内容与源一致
  - [正向] 不存在对象报错清晰（含 object key）
  - [正向] 变量路径正确解析
  - [负向] 无写权限时给出权限错误
  - [状态] 若网络偶发失败——标记为 flaky
