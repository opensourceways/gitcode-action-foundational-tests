标题:      API 限流与一致性——触发限流后恢复期内查询仍不丢数据

前置条件:
  - 仓库已启用 Actions 且存在至少 1 个已完成或运行中的 workflow run
  - 具备让 API 触发限流的能力（如短时间内大量请求）

操作步骤:
  1. 以超过平台限流阈值的高频（如 50 QPS）请求 runs API，持续 5 秒，触发 429
  2. 等待 Retry-After 指定的恢复时间
  3. 恢复正常速率（5 QPS）查询 runs 与 jobs API
  4. 验证恢复后的数据完整性与一致性

预期结果:
  - 高频阶段收到 429 响应
  - 恢复阶段的所有请求均成功（200）且数据完整
  - 无 run 或 job 数据在高频/恢复过程中损坏或丢失

验证点:
  - [正向] 高频阶段出现 429 响应
  - [正向] 恢复后所有查询返回 200
  - [正向] 恢复后数据与触发限流前一致

清理:      无（只读操作）
