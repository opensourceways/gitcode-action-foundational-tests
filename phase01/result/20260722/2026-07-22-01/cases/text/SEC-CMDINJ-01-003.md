母意图:    —
标题:      分支名不可直接插进 run 脚本导致命令注入

前置条件:
  - 仓库存在 workflow 监听 push 事件

操作步骤:
  1. 推送一个分支，其名称包含 shell 元字符（如分号、反引号、管道符）
  2. 触发 push workflow，该 workflow 将分支名通过表达式插入 run 脚本
  3. 观察 workflow 执行结果与日志

预期结果:
  - 分支名中的特殊字符被转义或表达式求值结果以字符串字面量形式传入
  - 系统不应执行分支名中嵌入的命令
  - 运行日志不暴露内部环境信息

验证点:
  - [负向] 未因分支名内容执行非预期命令
  - [负向] 日志中不出现因命令注入导致的异常系统输出
  - [正向] workflow 正常完成或按预期失败（非注入导致的任意执行）

清理:      重置 fixture 仓库
