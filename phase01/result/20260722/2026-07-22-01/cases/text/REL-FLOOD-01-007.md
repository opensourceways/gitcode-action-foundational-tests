母意图:    —
标题:      并发洪泛——同一仓库 50 个 push 同时触发应正确排队/限流不崩溃

前置条件:
  - 仓库已启用 Actions
  - 具备同时发起 50 个 push 的自动化能力
  - runner 资源总量不足以同时运行 50 个 job（存在排队压力）

操作步骤:
  1. 准备 50 个不同的 commit
  2. 在极短时间窗口内（< 1 秒）将 50 个 commit 同时 push 到同一分支
  3. 持续通过 API 查询 runs 状态，观察排队与限流行为
  4. 等待所有运行进入终态（最长等待 10 分钟）

预期结果:
  - 系统不崩溃、不 500、不丢失 runs
  - 超出并发上限的 runs 进入 queued 状态并被依次调度
  - 最终 50 个 run 全部完成（成功或失败），无 stuck 死锁

验证点:
  - [正向] runs 总数等于 50
  - [正向] 至少部分 run 曾经历 queued 状态（证明排队生效）
  - [负向] 无 run 卡在 queued 超过 10 分钟（死锁检测）
  - [负向] API 查询不返回 500 错误

清理:      重置 fixture 仓库
