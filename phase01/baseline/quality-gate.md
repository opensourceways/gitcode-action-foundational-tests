# 质量门禁（定义「凭什么敢上线」）

> L0 基线之一。定义分维度阈值与 blocker 判定规则，是第二部分报告分维度聚合与门禁判断的依据。
> **本文件是模板+示例，请按项目实际调整阈值。**

## 分维度门禁

| 维度 | 上线阈值（示例，待定） | blocker 判定 |
|---|---|---|
| 完备性 | Parity Matrix 中 P0 能力项 100% 有用例且通过；无「未知」阻塞项 | 任一 P0 能力项不支持或行为错误 |
| 兼容性 | 所有一致性用例（应与 GitHub 一致者）通过；差异确认用例均已文档化 | 存在未声明的破坏性差异 |
| 稳定性 | P0 稳定性用例全过；故障注入均满足恢复预期 | 故障后无法恢复 / 数据损坏 |
| 安全性 | **全部 negative 断言必须通过，零容忍** | 任一 secret 泄露 / 权限越界 / 注入成功 |
| 易用性 | P0/P1 错误信息类用例达标；核心迁移路径可跑 | 核心迁移路径完全不可用且无提示 |

## 全局 blocker 规则
1. 任一安全 `negative` 断言失败 → 直接 blocker，不可上线。
2. 任一风险登记册 blocker 风险项无 P0 用例覆盖 → 门禁不通过（覆盖盲区）。
3. DoD 未全绿 → 该批次不予交付第二部分。

## 判定输出约定
第二部分按维度聚合用例结果，对照本门禁产出「通过/不通过 + blocker 清单」。本部分只定义规则，不执行判定。

> 填写建议：阈值应与风险登记册 blocker 列表一致；安全维度保持零容忍。
