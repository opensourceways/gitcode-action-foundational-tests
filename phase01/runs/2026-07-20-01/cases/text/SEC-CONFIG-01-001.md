标题:      Secret 命名遵守约束校验：拒绝非法命名，接受合法命名

前置条件:
  - 具备创建 Secret 的仓库管理权限
  - 了解 Secret 命名约束：仅允许 [A-Z0-9_]+，不得以 ATOMGIT_ 或数字开头

操作步骤:
  1. 尝试创建名为 ATOMGIT_MY_TOKEN 的 Secret → 预期被拒绝
  2. 尝试创建名为 1MY_SECRET（数字开头）→ 预期被拒绝
  3. 尝试创建名为 my-token（含小写+特殊字符）→ 预期被拒绝
  4. 尝试创建名为 deploy.key（含点号）→ 预期被拒绝
  5. 创建名为 DEPLOY_KEY 和 NPM_TOKEN_2024 → 预期成功

预期结果:
  - 以 ATOMGIT_ 开头的名称被拒绝，有明确错误提示
  - 以数字开头的名称被拒绝
  - 含小写字母或特殊字符的名称被拒绝
  - 仅大写字母+数字+下划线的合法名称可创建成功

验证点:
  - [负向] ATOMGIT_ 开头 → 拒绝 + 错误提示
  - [负向] 数字开头 → 拒绝
  - [负向] 小写/特殊字符 → 拒绝
  - [正向] 合法名称 → 创建成功
