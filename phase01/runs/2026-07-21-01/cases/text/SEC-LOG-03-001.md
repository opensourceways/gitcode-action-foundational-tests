标题:      Workflow 日志的安全生命周期——保留期、下载权限

前置条件:
  - 存在可访问的 workflow 日志

操作步骤:
  1. 非项目成员尝试通过 API 下载日志
  2. 验证是否受权限控制
  3. 验证过期日志是否不可访问

预期结果:
  - 日志下载受权限控制
  - 过期日志不可访问

验证点:
  - [负向] 非项目成员不应能通过 API 下载日志
  - [负向] 超过保留期的日志不应可访问
  - [正向] 日志下载需 write 及以上权限
